关于 endless times 的诞生

从好久之前说起

得不到的永远在骚动，自从我小学期没有选上 python + QT，我就一直渴望拥有一个自己的博客。
原因有很多，一是我觉得这样很酷（这就是我的内心想法 咳咳），二是我发现学过的新东西确实需要做一些记录（让一年之后自己需要捡起来时可以迅速回忆），三是我真的很喜欢学了新东西和人分享（但是线下社交和微信显然都不能满足我的需求）

做一个吧！

这学期就已经尝试过一次了，但是由于我的 python 水平非常有限，最终还是没能做成。前几天凯文告诉我可以用 hexo + github 来做，而且比较简单，于是我大手一挥，开干！

《比较简单》

教程确实挺简单的，但是我的电脑貌似和教程没啥关系了TAT

记录几个卡了我很久的坑：

  • github 从2022年8月开始就已经取消了 github pages 里的 themes 设置，所以如果用 github pages 的静态网页来做的话，需要学习一下 jekyll ，我学习了大概3个小时，评价是“拉倒”。

  • hexo 确实整体来说比较简单，但是也有一个问题是，博客是链接在本地的服务器上的，所以所有修改都要在本地上进行再 push 上去，有一点点不是很方便呢

  • github 你为什么不让我 git clone 您

    • git clone from github 非常慢，所以大多时候会连接失败，或者报一个更奇怪的错误，譬如在 hexo init 的时候，他实际上是调用了 git clone ，而且加速和更改DNS都不是很管用，不过学会改 hosts 也是一种收获（雾）
    • 所以最后选择用国内的网站，用 Gitee 会快很多，而且直接从github拷仓库很快。

  • 管理员身份是个好身份，有两个奇怪的错误通过这个解决了

    • 一是 npm install 相关的报错，很怪！但是用管理员身份打开解决了！耶！
    • 二是 hexo d push到github的时候，ssh 出了一些奇怪的问题，同样，管理员！

还会长大

忘记是谁和我讲过，代码能跑的时候就不要动了。
果然，我试图向模板里加入插件的时候，出了一些错误，苦看3个小时，看不太懂。所以博客现在还是非常简陋的呀，等下一次闲一点的时候我会做一下美化和修改的！

最后
hello world!
